# SDLC Artifacts

The requirements, planning, design, traceability, and process artifacts developed for
AstraNotes across the quarter, in roughly chained order:

1. **Requirements** — `InitialRequirementSet.md` / `submission-InitialRequirementSet.md`
   → `RefinedRequirementBaselineforAstraNotes.md` / `submission-RefinedRequirements.md`
2. **Planning** — `BacklogandSprintZeroPlanforAstraNotes.md` /
   `submission-BacklogAndSprintZero.md`; `WorkingAgreementAstraNotes.md` /
   `submission-WorkingAgreement.md`; `DefinitionofDone.md` / `submission-DefinitionOfDone.md`
3. **Architecture & UML** — `ArchitectureDecisionLog.md` /
   `submission-ArchitectureDecisionLog.md`; `CompleteUMLDesignPackage.md` /
   `submission-UMLDesignPackage.md`
4. **Traceability** — `Requirements-to-UMLTraceabilityMatrix.md` /
   `submission-TraceabilityMatrix.md` (and the live code↔test map in
   [`../traceability.md`](../traceability.md))
5. **Design → implementation** — `ConnectingDesignToPrototypeOrImplementation.md` /
   `submission-ConnectingDesignToPrototype.md`
6. **Testing** — `submission-ShiftLeftTesting.md`, `submission-TestImprovementLog.md`
7. **Process / case studies** — `submission-CollaborativeGitWorkflow.md`,
   `submission-WaterfallChart.md`, `submission-FBISentinelViewpoint.md`

The working application in `src/` realizes these artifacts; see the repository
[`README.md`](../../README.md) and [`../traceability.md`](../traceability.md) for the
end-to-end mapping.
