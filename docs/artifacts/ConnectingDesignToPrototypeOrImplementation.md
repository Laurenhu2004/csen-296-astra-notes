# Week 6 Lab: AstraNotes Development Environment and First Realization Slices

For this lab, I want you to move from validated AstraNotes design into real development work with GitHub Copilot support.

## What To Submit
Submit one PDF or one text entry containing:
- your chosen development direction: programming language and desktop or web app decision
- a short description of your project structure and build or run environment
- a short description of your UI shell (menu, profile, settings, notes workspace)
- one or two first functionality slices you implemented or drafted
- a short traceability note showing which requirement IDs and UML elements those slices realize
- a short AI reflection describing how Copilot helped and what you changed

## What I Expect You To Build This Week
- a repository structure such as `src/`, `test/`, `docs/`, and build files
- a basic runnable or defensible application shell
- menu / profile / settings / notes layout decisions
- one or two first functions such as note creation or note listing
- which slice or slices you reviewed
- what weaknesses you found in readability, maintainability, responsibility clarity, or failure handling
- how you improved the slice
- what debugging or quality lesson you learned

## AI Use Guidance
You may use GitHub Copilot to help generate scaffolding, module outlines, boilerplate, and first-pass implementation slices. However, I want the final structure and choices to remain aligned with your validated AstraNotes artifacts.
